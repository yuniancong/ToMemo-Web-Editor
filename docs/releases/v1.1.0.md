# ToMemo Web Editor v1.1.0

`v1.1.0` 是首个以稳定桌面工作流为目标的功能版本，重点解决 ToMemo 配置在网页显示、批量生成、手动排序与手机端回导之间的一致性。

## 核心更新

### 可持久化的 Memo 拖拽排序

- 支持将单条 Memo 拖到任意两条内容之间。
- 支持将 Command/Shift 多选后的 Memo 作为一个有序分组拖动。
- 拖动过程中显示蓝色插入线和定位圆点，明确表示松开后的目标位置。
- 排序不仅更新页面状态，还会写入最终 ToMemo 配置。
- 同时同步 `notes` 数组和阶梯时间，保证 ToMemo 手机端按最新时间排序后与网页顺序一致。
- 只修改当前分类，其他分类及其 Memo 保持不变。

### 统一的 JSON 导入体验

- “粘贴 JSON”自动识别完整 ToMemo 配置和 `tomemo-content-package` AI 内容包。
- AI 内容包无需预先创建工作区；应用会自动建立空白 ToMemo `1.0` 配置并进入预览。
- JSON 解析和结构错误提供明确反馈。

### 更符合预期的数据操作

- 删除分类时级联删除其中全部 Memo，不再隐式移动到其他分类。
- AI 批量导入为新 Memo 生成稳定的阶梯时间，保持预览顺序。
- 文本编辑器中的 `Command+A` 恢复标准全选文本行为；只有页面区域使用 Memo 全选。

## 可靠性与兼容性

- AI 建议颜色经过严格 `RRGGBBAA` 校验，无效值回退到已验证默认色。
- 未验证的 ToMemo 配置版本保持只读，避免在未知 schema 上产生破坏性写入。
- Service Worker 对同源构建资源执行运行时缓存，提升离线加载可靠性。
- 未知配置字段继续原样保留。

## 验证结果

- 自动化测试：20 项全部通过。
- TypeScript 与 Vite 生产构建通过。
- 浏览器级拖拽验证通过，包括插入指示器与顺序持久化。
- 真实 ToMemo `1.0` 配置解析与无损往返通过。
- 真实手机 ToMemo 导入验证通过：新 ID 新增，相同 ID 覆盖，既有数据不受影响。

## 安装

### 从源码运行

```bash
git clone https://github.com/yuniancong/ToMemo-Web-Editor.git
cd ToMemo-Web-Editor
npm install
npm run dev
```

### 下载发布包

下载本 Release 的 `ToMemo-Web-Editor-Web.zip`，通过静态服务器运行其中的生产构建。

ChatGPT Projects 用户可同时下载 `ToMemo-ChatGPT-Projects-Knowledge.zip`。

## 升级说明

旧版本生成的配置不会自动重新排序。若需要手机端顺序与网页一致，请使用 `v1.1.0` 重新导入内容包或手动拖动并重新导出配置。
