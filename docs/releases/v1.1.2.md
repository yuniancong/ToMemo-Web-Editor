# ToMemo Web Editor v1.1.2

提升 AI 生成内容包的 JSON 容错能力，重点处理 shell 命令路径中的双引号转义问题。

## 问题背景

部分模型会输出如下内容：

```text
"content": "cd "/Users/name/Desktop""
```

路径两侧的双引号没有转义，因此不是合法 JSON，标准解析器会在对应位置报错。

## 本版本改进

### 导入器自动修复

- 严格 JSON 解析失败后进入受限修复流程。
- 自动识别 `title`、`content` 字段内部常见的未转义双引号。
- 修复后重新执行 JSON 解析和完整 AI 内容包 schema 校验。
- 无法安全修复时仍返回原始精确错误，不静默接受不确定数据。

### ChatGPT Projects 知识库强化

- 明确要求 JSON 字符串内部双引号写成 `\"`。
- 增加 shell 路径的正确与错误对照示例。
- 自检清单增加引号、反斜线和换行转义检查。

## 正确格式

```json
{
  "title": "桌面",
  "content": "cd \"/Users/name/Desktop\""
}
```

## 兼容性

- 不改变 `tomemo-content-package` schema。
- 不改变 ToMemo `1.0` 导出结构。
- 可直接从 `v1.1.x` 升级。

## 验证

- 全量自动化测试通过。
