# ToMemo Web Editor v1.1.1

修复搜索与临时排序视图中 Memo 序号语义的维护版本。

## 修复内容

此前搜索会根据筛选结果重新从 `01` 编号，使用户无法判断命中项在完整分类中的真实位置。

本版本将序号定义为 Memo 在当前分类“原始顺序”中的固定位置：

- 搜索只筛选显示内容，不重新编号。
- 例如原始第 `03`、`08`、`15` 条命中时，结果仍显示 `03`、`08`、`15`。
- 按标题或时间临时查看时同样保留原始序号。
- 只有完成手动拖拽并真正改写配置顺序后，序号才重新计算。

## 用户影响

- 搜索结果可直接映射回完整分类位置。
- 临时排序不再造成“配置顺序已改变”的错觉。
- 拖拽持久化排序与固定序号语义保持一致。

## 兼容性

- 不改变 ToMemo JSON schema。
- 不修改现有 Memo 数据或时间。
- 可直接从 `v1.1.0` 升级。

## 验证

- 自动化测试：21 项全部通过。
- 新增搜索命中项保持原始序号的浏览器行为测试。
- TypeScript 与 Vite 生产构建通过。
