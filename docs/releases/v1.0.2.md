# ToMemo Web Editor v1.0.2

修复桌面文本编辑器快捷键作用域的维护版本。

## 修复内容

应用使用 `Command+A` 作为当前 Memo 列表的全选快捷键，但 `v1.0.1` 错误地在整个应用范围拦截该组合键，导致 JSON、标题和正文编辑框无法使用浏览器标准的“全选文本”。

本版本重新定义快捷键作用域：

- 焦点位于 `input`、`textarea`、`select` 或 `contenteditable` 时，`Command+A` 保持标准文本全选行为。
- 焦点位于普通页面区域时，`Command+A` 才用于选择当前分类中的 Memo。
- Windows/Linux 的 `Ctrl+A` 遵循相同规则。

## 用户影响

- JSON 内容包可使用键盘快速全选、替换和粘贴。
- Memo 标题与正文编辑恢复原生桌面操作体验。
- Memo 批量选择快捷键继续保留。

## 兼容性

- 不改变 ToMemo 配置或 AI 内容包格式。
- 不迁移本地工作区数据。
- 可直接从任何 `v1.0.x` 版本升级。

## 验证

- 自动化测试：19 项全部通过。
- 增加文本输入目标不阻止 `Command+A` 默认行为的回归测试。
- TypeScript 与 Vite 生产构建通过。
