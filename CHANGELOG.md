# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/)。

## [1.1.2] - 2026-07-12

### Added

- AI 内容包解析增加 JSON 容错修复，可处理 shell 路径中常见的未转义双引号。

### Changed

- 强化 ChatGPT Projects 知识库中的 JSON 字符串转义规则、正确示例和自检条目。

### Validation

- 增加包含 `cd "/Users/..."` 非法原始输出的回归测试。
- 自动化测试和生产构建通过。

## [1.1.1] - 2026-07-12

### Fixed

- 搜索结果保留 Memo 在当前分类原始顺序中的固定序号，不再从 `01` 重新编号。
- 按标题或时间临时查看时同样保留原始序号；只有持久化拖拽排序后才重新计算。

### Validation

- 自动化测试：21 项全部通过。
- TypeScript 与 Vite 生产构建通过。

## [1.1.0] - 2026-07-12

### Added

- Memo 手动拖拽排序，支持在相邻内容之间显示插入线和定位圆点。
- 多选 Memo 作为一个有序分组拖动。
- 排序事务同时更新 `notes` 数组和 ToMemo 时间排序键，保证手机端顺序与网页一致。
- “粘贴 JSON”自动识别完整 ToMemo 配置和 AI 内容包。
- 无原配置时，从 AI 内容包自动创建空白 ToMemo 工作区。

### Changed

- 删除分类现在级联删除其全部 Memo，不再默认移动到其他分类。
- AI 批量导入使用阶梯时间，确保列表顺序可预测。
- 完整重写开源 README，增加数据模型、兼容性、工作流与技术设计说明。

### Fixed

- 修复文本编辑区 `Command+A` 被全局 Memo 多选快捷键拦截的问题。
- 修复 AI 内容包误放入完整配置入口时产生难以理解的结构错误。
- 修复 AI 建议颜色可能产生非法 ToMemo `colorAsHex` 的问题。
- 修复 Service Worker 未缓存构建资源导致离线加载不可靠的问题。

### Validation

- 真实 ToMemo `1.0` 配置解析和无损往返。
- 真实手机导入成功；新 ID 新增、相同 ID 覆盖。
- 自动化测试与生产构建通过。

## [1.0.0] - 2026-07-12

首个公开版本：提供 ToMemo 配置导入、三栏编辑、AI 内容包桥接、本地自动保存、结构校验、PWA 和 ChatGPT Projects 知识库。
