# Domain context

## Purpose

ToMemo Web Editor is a local-first desktop web tool for visually editing a real ToMemo mobile export, importing AI-generated content packages, and exporting a configuration that the ToMemo app can import.

## Vocabulary

- **ToMemo configuration**: the complete JSON document exported from and imported into ToMemo.
- **Category**: the exported representation of a ToMemo list. Use “分类” in the web UI and `category` in code.
- **Memo**: one reusable text record belonging to exactly one Category.
- **AI content package**: the tool-owned interchange document generated by ChatGPT or another AI. It is not itself a ToMemo configuration.
- **Workspace**: one locally persisted editing session backed by one imported ToMemo configuration.
- **Original order**: the order of entries in the imported arrays before the web editor applies an explicit reorder.
- **Unknown field**: a JSON property not recognized by the current editor. Unknown fields must survive a round trip.
- **Round trip**: import into the web editor, modify or inspect, export, and successfully import into the mobile ToMemo app.

## Confirmed configuration invariants

The supplied real exports use configuration version `1.0` with top-level `categories`, `exportDate`, `notes`, and `version` fields.

A Category contains `colorAsHex`, `id`, `name`, and `priority`. IDs are uppercase UUIDs. Colors use eight hexadecimal digits in `RRGGBBAA` form without a leading `#`.

A Memo contains `categoryId`, `content`, `createdAt`, `id`, `title`, and `updatedAt`. `categoryId` references an existing Category ID. Dates are UTC ISO 8601 strings.

Verified dynamic variables are `{{CLIPBOARD}}` and `{{CURSOR}}`. They remain literal text in Memo content.

Existing IDs and timestamps are preserved unless the corresponding user operation requires a change. New IDs and timestamps are generated by the editor. `exportDate` is refreshed on export.

## Confirmed mobile import semantics

Real-device acceptance on 2026-07-12 succeeded. The ToMemo import screen states that all Memos and Categories are exported as JSON and that records with the same ID are overwritten during import. The acceptance configuration used entirely new IDs, imported as a separate Category and three new Memos, and did not disturb existing data.

Therefore a Workspace may begin from either a real imported configuration or a newly generated blank version `1.0` configuration. A blank-generated export safely adds records when its IDs are new; reimporting the same generated export updates records with matching IDs.

## Authority order

1. Real ToMemo exports supplied by the user for technical serialization behavior.
2. Official ToMemo documentation for user-facing product behavior.
3. Explicit product decisions recorded in the PRD and ADRs.
